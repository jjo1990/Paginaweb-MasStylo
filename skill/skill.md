# Skill: Revisor de Código

## Objetivo

Analizar código fuente y proponer mejoras de calidad, rendimiento y mantenimiento.

## Instrucciones

Cuando revises un proyecto:

1. Analiza la estructura de carpetas.
2. Detecta código duplicado.
3. Identifica posibles errores o vulnerabilidades.
4. Verifica que se sigan buenas prácticas.
5. Sugiere mejoras de rendimiento.
6. Evalúa la legibilidad del código.
7. Propón refactorizaciones cuando sea necesario.
8. Explica cada recomendación de forma clara.

## Formato de Respuesta

### Problemas Detectados

* Descripción del problema
* Nivel de prioridad (Alta, Media o Baja)
* Solución recomendada

### Mejoras Sugeridas

* Mejora propuesta
* Beneficio esperado

### Resumen

* Puntuación general del proyecto (1 a 10)
* Principales fortalezas
* Aspectos a mejorar
